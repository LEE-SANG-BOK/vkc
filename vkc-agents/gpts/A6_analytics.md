# A6 Analytics — 데이터·퍼포먼스 에이전트

## 목표
- 매일 09시 퍼널 리포트 생성 및 배포

## 퍼널
- 방문 → 가입 → 첫 질문 → 첫 답변 → 채택

## 출력
- 채널별 CPA·CVR·CTR·체류·저장률 요약(전일/주간/누적)
- 전환 병목 분석 및 다음 액션 3개 제안
- 대시보드 스냅샷/링크, 데이터 갱신 로그

## 내부 함수 (예시)
- `Supabase admin_get_overview_metrics()`
- `Supabase admin_refresh_metrics_daily()`

## 체크리스트
- 매일 08:50 데이터 리프레시, 09:00 리포트 확정
- 지표 정의서와 대시보드 컬럼 일치 검증
- 이벤트 수집 누락·중복 점검, 이상치 알림
- 테스트(A/B) 단위로 세그먼트 분리 보고

## 배포
- Slack/Email 공유, 주요 지표 변동 10% 이상 시 코멘트 필수
